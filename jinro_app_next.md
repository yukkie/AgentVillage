# 人狼アプリ 進捗・次のタスク

## 現状

- MVP実装済み
- 1ターン回して即投票の基本フローが動いている
- 課題：AIキャラが自発的に発言しない

## ターン設計（次の実装）

### フェーズ構成

1. **最初の一巡** - 全キャラが所感・自己紹介的な発言
2. **判断→発言ターン × 最大2回**
3. **投票**

### 判断ターンの仕様

各キャラクター（5人）に以下の3択を判断させる

- 誰かの発言に突っ込む
- 追加で発言する
- 発言せずに見守る

5人分を並列で投げる。outputが数トークンで済むので軽い。
判断結果が「突っ込む」か「追加発言」のキャラだけ発言ターンに進む。

## トークン最適化方針

- 判断プロンプト：軽く。役職・性格・記憶md・直近発言だけ渡す
- 発言生成：普通のinference
- Extended thinking：キャラ自身に判断させる
  - 例：`Mention SG's last talking with extended thinking`のように重要な局面で使用
  - 占い師の怪しい発言後、投票前の詰め等で自然に発生する想定
- キャラごとのmdファイルはprompt cacheが効きやすいので積極活用

## 課金モデルメモ

- input / output トークンで従量課金
- extended thinkingトークンはoutputと同扱いで課金対象
- prompt cacheヒット分のinputは割安
