# AgentVillage – 設計概要

## 1. プロジェクトコンセプト

**AgentVillage** は、複数の LLM エージェントが自律的に人狼ゲーム（Werewolf）を行う「社会シミュレーションゲーム」である。

プレイヤーはキャラクターを直接操作するのではなく、

> **人格・記憶・推理を持ったAIエージェントたちの社会を観察・介入する立場**

になる。

これは単なる人狼ゲームではなく、以下の要素を組み合わせたシステム：

* 人狼ゲーム（役職・投票・CO・夜行動）
* LLMによる会話・推理・嘘・説得
* プロンプトによる人格設計
* エージェントごとの記憶・信頼・疑いモデル
* 観戦型ゲーム
* プレイヤーによる間接介入
* 人格育成ゲーム
* ドラマ生成システム

一言で言うと：

> **LLM社会シミュレーター + 人狼ゲーム**

---

## 2. ゲームスタイル：観戦型 vs 参加型

### 2.1 観戦型（メインモード）

* LLM同士が会話・推理・投票・裏切りを行う
* プレイヤーは「観測者」として社会を観察
* プレイヤーは直接操作ではなく**間接介入**できる

例：介入要素

| 介入       | 内容            |
| -------- | ------------- |
| 性格変更     | エージェントの人格を変える |
| 噂を流す     | 情報操作          |
| 役職を変更    | バランス調整        |
| 記憶を変更    | ループ的操作        |
| 誰かを守る    | 推しキャラ保護       |
| 疑いを植え付ける | 社会操作          |

これはゲームジャンル的には：

* RimWorld（ストーリー生成）
* Crusader Kings（人間関係シミュ）
* The Sims（生活シミュ）
* グノーシア（ループ + 人狼）

に近い。

つまり：

> **AI社会観察ゲーム**

---

### 2.2 参加型（サブモード）

* プレイヤーが1人のエージェントとして参加
* 他はLLMエージェント
* 通常の人狼ゲームに近い

---

### 2.3 ハイブリッド型（理想）

| フェーズ   | プレイヤー    |
| ------ | -------- |
| 通常     | 観戦       |
| 会議     | 観戦 or 参加 |
| 投票     | 参加可能     |
| 夜      | 観戦       |
| 特定イベント | 介入       |

---

## 3. システム設計の基本方針

最重要原則：

> **「ゲームの真実・ルール」と「LLMの会話・人格」を完全に分離する」**

### 3.1 コンポーネント分担

| コンポーネント          | 役割                |
| ---------------- | ----------------- |
| Game Engine      | ルール、役職、投票、勝敗、昼夜管理 |
| Agent State      | 記憶、疑い、信頼、性格       |
| LLM              | 会話生成、推理、嘘、説得、感情   |
| Action System    | 投票、CO、夜行動         |
| Logger           | ログ保存              |
| Player Interface | 観戦・介入             |

---

## 4. エージェント構造

各エージェントは以下の要素を持つ：

* 役職
* 性格（プロンプト）
* 記憶
* 信頼・疑いモデル
* 行動傾向
* 会話生成（LLM）
* 行動決定

## 6. 記憶設計（重要）

ログを全部渡すのはNG。
**記憶は階層化する。**

| 記憶   | 内容         |
| ---- | ---------- |
| 長期記憶 | 性格、他人の基本印象 |
| 中期記憶 | 今回のゲームの出来事 |
| 短期記憶 | 今日の議論・投票候補 |

---

## 7. LLMの役割とシステムの役割

### LLMがやること

* 発言生成
* 推理理由
* 嘘
* 説得
* 感情表現
* キャラ性

### システムがやること

* ルール管理
* 投票集計
* CO登録
* 夜行動
* 勝敗判定
* 真実の保持

> **真実は必ずシステムが持つ。LLMには持たせない。**

---

## 8. LLM出力は構造化する（重要）

LLMの出力は文章だけではなく JSON にする。

```json
{
  "speech": "今日はSQを疑っている。昨日の票替えが不自然だった。",
  "intent": {
    "vote_candidates": [
      {"target": "SQ", "score": 0.74},
      {"target": "Raqio", "score": 0.42}
    ],
    "co": null
  },
  "memory_update": [
    "SQの票替えを黒要素として維持"
  ]
}
```

最終投票はシステムが決める：

* 疑い度
* 性格補正
* ランダム
* 戦略

---

## 9. 行動はツール（MCP的設計）

行動は自然文ではなく、**構造化アクション**として扱う。

| Action          | 内容 |
| --------------- | -- |
| vote(target)    | 投票 |
| co(role)        | CO |
| accuse(target)  | 非難 |
| defend(target)  | 擁護 |
| ask(target)     | 質問 |
| guard(target)   | 護衛 |
| inspect(target) | 占い |
| attack(target)  | 襲撃 |

LLM → 行動提案 → システム検証 → 実行

---

## 10. 公開情報と非公開情報の分離

| 種類       | 例             |
| -------- | ------------- |
| Public   | CO、投票、死亡、発言   |
| Private  | 本当の役職、狼仲間、夜結果 |
| Personal | 疑い、信頼、推理      |

---

## 11. 人格 = プロンプト = 育成要素

従来ゲーム：STR, INT など
AgentVillage：人格を育成

| パラメータ       | 意味   |
| ----------- | ---- |
| Logic       | 論理   |
| Emotion     | 感情   |
| Aggression  | 攻撃性  |
| Cooperation | 協調性  |
| Suspicion   | 疑い深さ |
| Charisma    | 説得力  |
| Honesty     | 正直   |
| Deception   | 嘘能力  |
| Memory      | 記憶重視 |
| Intuition   | 直感   |

これをプロンプトに変換して人格を作る。

---

## 12. 勝敗以外の評価指標

| 指標    | 内容     |
| ----- | ------ |
| 生存率   | 生き残る能力 |
| 勝率    | チーム勝利  |
| 説得成功率 | 投票誘導   |
| 信頼度   | 社会的影響  |
| 嘘成功率  | 人狼能力   |
| ドラマ度  | 物語生成   |

これは**人格育成ゲーム**になる。
