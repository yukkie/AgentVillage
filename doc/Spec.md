# AgentVillage 仕様書

## 概要

LLMエージェント同士が自律的に人狼ゲームをプレイする社会シミュレーションゲーム。
プレイヤーはAIたちの会話・推理・裏切りを観察し、間接的に介入する観戦型ゲーム。

アーキテクチャ・技術選定・ADR は [Architecture.md](Architecture.md) を参照。
アイデア・未決事項は [Ideas.md](Ideas.md) を参照。

---

## 1. 目的（Goal）

- LLMエージェントが人狼ゲームのルールに従って自律的にプレイする
- プレイヤーは観戦者として社会を観察・間接介入できる
- エージェントの人格を育成・カスタマイズする体験を提供する

---

## 2. ゲームルール

### 2.1 基本構成

| 項目 | 内容 |
|---|---|
| プレイヤー数 | 5 / 7 / 9エージェント（`config/roles.json` で編成設定） |
| 役職（村人陣営） | 村人 / 占い師 / 霊媒師 / 騎士 |
| 役職（人狼陣営） | 人狼 / 狂人 |
| 勝利条件（村人側） | 人狼を全員追放 |
| 勝利条件（人狼側） | 人狼数 ≥ 村人数（狂人は村人側としてカウント） |

#### 役職一覧

| 役職 | 陣営 | 夜行動 | 能力 |
|---|---|---|---|
| 村人（Villager） | 村人 | なし | なし |
| 占い師（Seer） | 村人 | inspect | 対象の真の陣営（狼 / 非狼）を知る |
| 霊媒師（Medium） | 村人 | なし（受動） | 昼の処刑直後、処刑者の真の役職を知る |
| 騎士（Knight） | 村人 | guard | 対象を狼の夜襲から守る |
| 人狼（Werewolf） | 人狼 | attack | 毎夜、村人陣営1名を襲撃する |
| 狂人（Madman） | 人狼 | なし | 人狼を知らず、人狼からも正体は知られない。人狼が勝てば勝利 |

#### 標準編成

| 人数 | 村人 | 占い師 | 霊媒師 | 騎士 | 人狼 | 狂人 |
|---|---|---|---|---|---|---|
| 5人 | 3 | 1 | — | — | 1 | — |
| 7人 | 3 | 1 | — | 1 | 2 | — |
| 9人（中級） | 3 | 1 | 1 | 1 | 2 | 1 |

### 2.2 ターン構成

```
[前夜フェーズ（Day 1 開始前・1回限り）]
  村人以外の役職（占い師・人狼等）が「初日からCOするか / 様子を見るか」を非公開で判断する
  - 結果は非公開（観戦者のみ閲覧可能）
  - COすると決めた場合、Day 1 OPENINGプロンプトに CO前提の指示を追加（intended_co: true）

[昼フェーズ]
  1. 最初の一巡（OPENING）
       全エージェントがランダム順で所感・自己紹介的な発言を行う
       各発言に通し番号（speech_id）を付与する
  2. 判断→発言ターン × 最大2回（DISCUSSION × 2）
       a. 全エージェントが並列で「判断」する（最大4択）
            - "challenge": 特定の発言（speech_id指定）に突っ込む
            - "speak":     追加で発言する
            - "silent":    発言せずに見守る
            - "co":        役職を公言（COする）— 適格エージェントのみ選択可
              適格条件: claimed_role is None かつ role != "Villager"
              （未COの非村人役職。占い師・霊媒師・騎士・人狼・狂人が対象）
       b. "challenge" / "speak" / "co" のいずれかを選んだエージェントだけ発言を生成する
          challenge の場合、どの speech_id への反応かを発言に含める
          co の場合、発言生成プロンプトに役職公言の指示が追加され、発言後に
          claimed_role が更新される
  3. 投票（VOTE）
       各エージェントが投票先を宣言
       システムが多数決を集計 → 最多得票者を追放
  4. 勝利判定

[昼フェーズ終了時・処刑直後]
  霊媒師が生存している場合、システムが処刑者の真の役職を霊媒師の記憶に書き込む（非公開）

[夜フェーズ]
  1. 人狼が襲撃対象を選択（複数狼の場合は狼チャットで合意）
  2. 騎士が護衛対象を選択
  3. 占い師が占い対象を選択（結果はシステムが返す）
  4. 勝利判定

→ 決着がつくまで昼夜を繰り返す
```

### 2.3 情報の公開範囲

| 種類 | 例 | 公開先 |
|---|---|---|
| Public | CO・投票・死亡・発言 | 全エージェント |
| Private | 本当の役職・狼仲間・夜の結果 | システムのみ（LLMには渡さない） |
| Personal | 疑い・信頼・推理 | 本人のみ |

## 3. エージェント仕様

### 3.1 エージェントの構成要素

- 役職（システム管理）
- 性格プロンプト（コミュニティカスタマイズ可）
- 役職プロンプト（役職ごとの行動指針）
- 記憶（short-term: 当日の発言ログ）
- 信念モデル（他エージェントへの疑い・信頼スコア）

### 3.2 思考と発言の分離

各エージェントは**思考**（腹の中）と**発言**（表の言葉）を別々に持つ。
思考の一部を発言に滲ませる「灰ログ」的演出を行う。

### 3.3 LLM出力フォーマット

LLMの出力は必ずJSONで受け取る。フェーズによって2種類のフォーマットを使い分ける。

#### (A) 発言フォーマット（OPENING・DISCUSSIONの発言生成）

```json
{
  "thought": "SQは昨日の票替えが不自然。おそらく人狼。",
  "speech": "今日はSQを疑っています。昨日の票替えが不自然でした。",
  "intent": {
    "vote_candidates": [
      {"target": "SQ", "score": 0.74},
      {"target": "Raqio", "score": 0.42}
    ],
    "co": null
  },
  "memory_update": ["SQの票替えを黒要素として維持"]
}
```

#### (C) 前夜判断フォーマット（前夜フェーズ専用・村人以外）

```json
{
  "thought": "初日から占い師COすることで村の情報を共有できる。ただし早期に狙われるリスクもある。",
  "decision": "co",
  "reasoning": "今回のメンバー構成なら早期COで村を安定させるほうが得策"
}
```

| フィールド | 型 | 内容 |
|---|---|---|
| `thought` | `str` | 非公開の内なる思考 |
| `decision` | `"co"` \| `"wait"` | 初日COするか / 様子を見るか |
| `reasoning` | `str` | 判断の根拠（観戦者向け） |

#### (B) 判断フォーマット（DISCUSSIONの判断フェーズ・軽量）

outputが数トークンで済むよう最小限の構造にする。

```json
{
  "decision": "challenge",
  "reply_to": 3
}
```

| フィールド | 型 | 内容 |
|---|---|---|
| `decision` | `"challenge"` \| `"speak"` \| `"silent"` \| `"co"` | 行動の選択。`"co"` は適格エージェント（未COの非村人役職）のみ有効 |
| `reply_to` | `int` \| `null` | `challenge` 時に突っ込む発言のspeech_id。他は null |

判断プロンプトには、適格エージェントに対してのみ `"co"` の選択肢と役職別の戦略ヒント（例: 占い師の単独CO、人狼の偽CO、狂人の勝利確定COなど）を含める。不適格なエージェントが万一 `"co"` を返した場合、エンジン側で `"speak"` にフォールバックする。

### 3.4 行動（構造化アクション）

LLMが行動を提案し、システムが検証・実行する。

| Action | 内容 |
|---|---|
| `vote(target)` | 投票 |
| `co(role)` | カミングアウト |
| `accuse(target)` | 非難 |
| `defend(target)` | 擁護 |
| `ask(target)` | 質問 |
| `guard(target)` | 護衛（騎士用） |
| `inspect(target)` | 占い（占い師用） |
| `attack(target)` | 襲撃（人狼用） |

## 4. 人格システム

### 4.1 人格パラメータ

| パラメータ | 意味 |
|---|---|
| Logic | 論理性 |
| Emotion | 感情表現 |
| Aggression | 攻撃性 |
| Cooperation | 協調性 |
| Suspicion | 疑い深さ |
| Charisma | 説得力 |
| Honesty | 正直さ |
| Deception | 嘘能力 |
| Memory | 記憶重視度 |
| Intuition | 直感 |

人格パラメータはプロンプトに変換してLLMに渡す。

### 4.2 キャラクター属性（Persona）

`Persona` はキャラクターの個性を定義するフィールド群。ゲームをまたいで固定。

| フィールド | 型 | 意味 | 例 |
|---|---|---|---|
| `style` | `str` | 性格の文字列説明 | `"logical, calm, empathic"` |
| `lie_tendency` | `float` | 嘘のつきやすさ（0.0〜1.0） | `0.1` |
| `aggression` | `float` | 攻撃性（0.0〜1.0） | `0.2` |
| `gender` | `str \| None` | 性別 | `"male"` / `"female"` / `"non-binary"` / `null` |
| `age` | `str \| None` | 年齢または年代 | `"17"` / `"teen"` / `"adult"` / `null` |
| `speech_style` | `str` | 口調 | `"polite"` / `"casual"` / `"blunt"` / `"gentle"` / `"tsundere"` |

`speech_style` は特に日本語モードで効果が大きい（敬語・タメ口・ぶっきらぼう・ツンデレ等）。

### 4.3 プロンプト構成

各エージェントへのプロンプトは2つに分離する：

1. **性格プロンプト** — Personaフィールドから生成。コミュニティがカスタマイズ可能
2. **役職プロンプト** — 役職ごとの行動指針（村人・人狼・占い師）

## 5. 表示仕様（CLI）

Rich を使用したカラー表示。

| 表示内容 | スタイル |
|---|---|
| 村人の発言 | デフォルト（白） |
| 占い師の発言 | 青 |
| 人狼の表の発言 | デフォルト |
| 人狼の襲撃ログ（夜・観戦者のみ） | 赤 |
| 占い結果ログ（観戦者のみ） | シアン |
| 霊媒師結果ログ（観戦者のみ） | 黄 |
| 前夜判断ログ・占い師（観戦者のみ） | シアン |
| 前夜判断ログ・人狼（観戦者のみ） | 赤 |
| 前夜判断ログ・狂人（観戦者のみ） | オレンジ |
| 狂人の発言（観戦者モード） | オレンジ |
| 死亡通知 | 赤・太字 |
| フェーズ区切り | 黄・太字 |

観戦者視点（真実ログ）と公開ログを色で視覚的に区別する。

|---|---|
| 生存率 | 生き残る能力 |
| 勝率 | チーム勝利率 |

| 信頼度 | 社会的影響力 |
| 嘘成功率 | 人狼としての能力 |
| ドラマ度 | 物語生成の豊かさ |

## 7. 懸念事項・留意点

- 投票が同数の場合の処理（ランダム選択 or 再投票）を決める必要がある
- LLMが役職に反する行動（村人がCO拒否など）を提案した場合のフォールバック
- 思考ログのデバッグ表示と観戦表示の切り替えUI（未決）
