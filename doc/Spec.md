# AgentVillage 仕様書

## 概要

LLMエージェント同士が自律的に人狼ゲームをプレイする社会シミュレーションゲーム。
プレイヤーはAIたちの会話・推理・裏切りを観察し、間接的に介入する観戦型ゲーム。

アーキテクチャ・技術選定・ADR は [Architecture.md](Architecture.md) を参照。
アイデア・未決事項は [Ideas.md](Ideas.md) を参照。

---

## 1. 目的（Goal）

- LLMエージェントが人狼ゲームのルールに従って自律的にプレイする
- プレイヤーは観戦者として社会を観察・間接介入できる
- エージェントの人格を育成・カスタマイズする体験を提供する

---

## 2. ゲームルール

### 2.1 基本構成（MVP）

| 項目 | 内容 |
|---|---|
| プレイヤー数 | 5エージェント固定 |
| 役職 | 村人×3 / 人狼×1 / 占い師×1 |
| 勝利条件（村人側） | 人狼を全員追放 |
| 勝利条件（人狼側） | 人狼数 ≥ 村人数 |

### 2.2 ターン構成

```
[昼フェーズ]
  1. 各エージェントが順番に発言（全員1回）
  2. 各エージェントが推理を宣言（誰を・なぜ疑っているか）
  3. 各エージェントが投票先を宣言
  4. システムが多数決を集計 → 最多得票者を追放
  5. 勝利判定

[夜フェーズ]
  1. 人狼が襲撃対象を選択
  2. 占い師が占い対象を選択（結果はシステムが返す）
  3. 勝利判定

→ 決着がつくまで昼夜を繰り返す
```

### 2.3 情報の公開範囲

| 種類 | 例 | 公開先 |
|---|---|---|
| Public | CO・投票・死亡・発言 | 全エージェント |
| Private | 本当の役職・狼仲間・夜の結果 | システムのみ（LLMには渡さない） |
| Personal | 疑い・信頼・推理 | 本人のみ |

## 3. エージェント仕様

### 3.1 エージェントの構成要素

- 役職（システム管理）
- 性格プロンプト（コミュニティカスタマイズ可）
- 役職プロンプト（役職ごとの行動指針）
- 記憶（short-term: 当日の発言ログ）
- 信念モデル（他エージェントへの疑い・信頼スコア）

### 3.2 思考と発言の分離

各エージェントは**思考**（腹の中）と**発言**（表の言葉）を別々に持つ。
思考の一部を発言に滲ませる「灰ログ」的演出を行う。

### 3.3 LLM出力フォーマット

LLMの出力は必ずJSONで受け取る。

```json
{
  "thought": "SQは昨日の票替えが不自然。おそらく人狼。",
  "speech": "今日はSQを疑っています。昨日の票替えが不自然でした。",
  "intent": {
    "vote_candidates": [
      {"target": "SQ", "score": 0.74},
      {"target": "Raqio", "score": 0.42}
    ],
    "co": null
  },
  "memory_update": ["SQの票替えを黒要素として維持"]
}
```

### 3.4 行動（構造化アクション）

LLMが行動を提案し、システムが検証・実行する。

| Action | 内容 |
|---|---|
| `vote(target)` | 投票 |
| `co(role)` | カミングアウト |
| `accuse(target)` | 非難 |
| `defend(target)` | 擁護 |
| `ask(target)` | 質問 |
| `guard(target)` | 護衛（騎士用） |
| `inspect(target)` | 占い（占い師用） |
| `attack(target)` | 襲撃（人狼用） |

## 4. 人格システム

### 4.1 人格パラメータ

| パラメータ | 意味 |
|---|---|
| Logic | 論理性 |
| Emotion | 感情表現 |
| Aggression | 攻撃性 |
| Cooperation | 協調性 |
| Suspicion | 疑い深さ |
| Charisma | 説得力 |
| Honesty | 正直さ |
| Deception | 嘘能力 |
| Memory | 記憶重視度 |
| Intuition | 直感 |

人格パラメータはプロンプトに変換してLLMに渡す。

### 4.2 プロンプト構成

各エージェントへのプロンプトは2つに分離する：

1. **性格プロンプト** — 人格パラメータから生成。コミュニティがカスタマイズ可能
2. **役職プロンプト** — 役職ごとの行動指針（村人・人狼・占い師）

## 5. 表示仕様（CLI）

Rich を使用したカラー表示。

| 表示内容 | スタイル |
|---|---|
| 村人の発言 | デフォルト（白） |
| 占い師の発言 | 青 |
| 人狼の表の発言 | デフォルト |
| 人狼の襲撃ログ（夜・観戦者のみ） | 赤 |
| 占い結果ログ（観戦者のみ） | シアン |
| 死亡通知 | 赤・太字 |
| フェーズ区切り | 黄・太字 |

観戦者視点（真実ログ）と公開ログを色で視覚的に区別する。

## 6. 勝敗以外の評価指標

| 指標 | 内容 |
|---|---|
| 生存率 | 生き残る能力 |
| 勝率 | チーム勝利率 |
| 説得成功率 | 投票誘導の成功度 |
| 信頼度 | 社会的影響力 |
| 嘘成功率 | 人狼としての能力 |
| ドラマ度 | 物語生成の豊かさ |

---

## 7. 懸念事項・留意点

- 投票が同数の場合の処理（ランダム選択 or 再投票）を決める必要がある
- LLMが役職に反する行動（村人がCO拒否など）を提案した場合のフォールバック
- 思考ログのデバッグ表示と観戦表示の切り替えUI（未決）
