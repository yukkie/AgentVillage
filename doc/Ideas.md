# AgentVillage — アイデア統合メモ

## 1. プロジェクトコンセプト

**AgentVillage** は、複数のLLMエージェントが自律的に人狼ゲームを行う「社会シミュレーションゲーム」。

プレイヤーはキャラクターを直接操作するのではなく、

> **人格・記憶・推理を持ったAIエージェントたちの社会を観察・介入する立場**

になる。

インスピレーション源：
- 人狼BBS の **灰ログ文化**（思考の一部が滲み出る演出）
- サカつく（育成しながら見守る）

一言で言うと：

> **AgentVillage = 人狼 + The Sims + RimWorld + グノーシア + LLM**

---

## 2. ゲームスタイル

### 2.1 観戦型（メインモード）

- LLM同士が会話・推理・投票・裏切りを行う
- プレイヤーは「観測者」として社会を観察
- **直接操作ではなく間接介入**できる

| 介入         | 内容              |
| ------------ | ----------------- |
| 性格変更     | エージェントの人格を変える |
| 噂を流す     | 情報操作          |
| 役職を変更   | バランス調整      |
| 記憶を変更   | ループ的操作      |
| 誰かを守る   | 推しキャラ保護    |
| 疑いを植え付ける | 社会操作      |

### 2.2 参加型（サブモード）

プレイヤーが1人のエージェントとして参加。他はLLMエージェント。

### 2.3 ハイブリッド型（理想）

| フェーズ       | プレイヤー        |
| -------------- | ----------------- |
| 通常           | 観戦              |
| 会議           | 観戦 or 参加      |
| 投票           | 参加可能          |
| 夜             | 観戦              |
| 特定イベント   | 介入              |

---

## 3. システム設計の基本方針

最重要原則：

> **「ゲームの真実・ルール」と「LLMの会話・人格」を完全に分離する」**

### 3.1 コンポーネント分担

| コンポーネント     | 役割                              |
| ------------------ | --------------------------------- |
| Game Engine (GM)   | ルール・役職・投票・勝敗・昼夜管理（**決定論的コード**） |
| Agent State        | 記憶・疑い・信頼・性格            |
| LLM                | 発言生成・推理・嘘・説得・感情    |
| Action System      | 投票・CO・夜行動                  |
| Logger             | ログ保存・リプレイ                |
| Player Interface   | 観戦・介入                        |

GMロジックはLLMの気まぐれに左右されない確実な進行管理として実装すること。

### 3.2 真実の所在

> **真実は必ずシステムが持つ。LLMには持たせない。**

| 種類     | 例                          |
| -------- | --------------------------- |
| Public   | CO・投票・死亡・発言        |
| Private  | 本当の役職・狼仲間・夜結果  |
| Personal | 疑い・信頼・推理            |

---

## 4. エージェント構造

各エージェントは以下の要素を持つ：

- 役職
- 性格（プロンプト）
- 記憶（長期・中期・短期）
- 信頼・疑いモデル
- 行動傾向
- 会話生成（LLM）
- 行動決定

---

## 5. プロンプト設計

各キャラクターへのプロンプトは **2つに分離**：

1. **性格プロンプト** — オープンにしてコミュニティがカスタマイズ可能
2. **役職プロンプト** — 村人・人狼・占い師等の役職ごとの行動指針

性格と役職を分離することで、同じ役職でも性格次第で戦略・発言が変わる。

---

## 6. 思考と発言の分離（灰ログ的演出）

- **思考**（腹の中）と**発言**（表の言葉）を別々に管理
- 思考過程の一部をあえて発言に滲ませる「灰ログ」的演出も検討
- 思考ログは観戦用・デバッグ用で表示モードを切り替え可能にする

## 8. 記憶設計

ログを全部渡すのはNG。**記憶は3層に分ける。**

| 層 | 名前 | 内容 | スコープ | 実装状態 |
| -- | ---- | ---- | -------- | -------- |
| 1 | skill_memory（スキル記憶） | 役職・戦略の汎用的な気づき（「人狼は序盤に疑惑を他者に向けると有効」等） | **ゲームをまたいで永続** | ❌ |
| 2 | episodic_memory（エピソード記憶） | 今ゲームの出来事・観察（「SQが怪しい」「GinaがCOした」等） | ゲームごとにリセット | ✅（現在の `memory_summary`） |
| 3 | short_term（短期記憶） | 今日の議論・発言ログ | ターンごとにリセット | ✅（`today_log` として実装済み） |

キャラクターごとに状態ファイルを管理し、ゲームのフェーズが進むたびに更新。

### skill_memory の実装方針（❌ 未実装）
- `AgentState` に `skill_memory: list[str]` フィールドを追加
- LLMの `memory_update` 出力を2種類に分ける（例: `episode_updates` / `skill_updates`）
- ゲーム開始時に `episodic_memory` はクリア、`skill_memory` は引き継ぐ
- ゲームを重ねるごとに洗練されていく（18番の自動要約とも連携。コスト増に注意）

## 9. LLM出力の構造化

LLMの出力は文章だけでなく **JSON** にする。

```json
{
  "thought": "SQは昨日の票替えが不自然。おそらく人狼。",
  "speech": "今日はSQを疑っています。昨日の票替えが不自然でした。",
  "intent": {
    "vote_candidates": [
      {"target": "SQ", "score": 0.74},
      {"target": "Raqio", "score": 0.42}
    ],
    "co": null
  },
  "memory_update": ["SQの票替えを黒要素として維持"]
}
```

`thought` フィールドが灰ログ演出の基盤となる。最終投票はシステムが決定（疑い度・性格補正・ランダム・戦略）。

## 13a. ターン設計（昼フェーズの会話ループ）

### フェーズ構成

1. **最初の一巡** — 全キャラが所感・自己紹介的な発言
2. **判断→発言ターン × 最大2回**
3. **投票**

### 判断ターンの仕様

各キャラクター（5人）に以下の **3択を並列で判断**させる：

- 誰かの発言に突っ込む
- 追加で発言する
- 発言せずに見守る

5人分を並列投げ。outputが数トークンで済むので軽い。
判断結果が「突っ込む」か「追加発言」のキャラだけ発言ターンに進む。

---

## 13b. トークン最適化方針

| フェーズ | 方針 |
| -------- | ---- |
| 判断プロンプト | 軽く。役職・性格・記憶md・直近発言だけ渡す |
| 発言生成 | 通常のinference |
| Extended thinking | キャラ自身に判断させる重要局面で使用。占い師の怪しいCO後・投票前の詰め等で自然に発生する想定 |

- Extended thinkingの例：`Mention SG's last talking with extended thinking`のように重要な局面で指示
- キャラごとのmdファイルはprompt cacheが効きやすいので積極活用

### 課金モデルメモ

- input / outputトークンで従量課金
- extended thinkingトークンはoutputと同扱いで課金対象
- prompt cacheヒット分のinputは割安

## 16. 次の大きな変更

### ✅ 16.1 前夜ターン（Day 1 開始前）

Day 1 のOPENING発言の前に、占い師・人狼だけが行う**非公開の判断フェーズ**を追加する。

- 占い師・人狼それぞれが「初日からCOするか / 様子を見るか」を判断する
- COすると決めた場合、Day 1 のOPENING発言プロンプトにCO前提の指示を加える（`intended_co: true`）
- 結果は非公開（観戦者のみ閲覧可能）

### ❌ 16.2 Discussion時のCOオプション追加

現在の判断3択（challenge / speak / silent）に **co** を追加する。

- 対象：まだCOしていない Villager 以外のエージェント（`claimed_role is None` かつ `role != "Villager"`）
- COを選んだ場合、発言生成プロンプトにCO宣言の指示を加える
- 発言後に `claimed_role` を更新（既存の intent.co ルートと統合する）

### ❌ 16.3 夜の狼同士の会話（7人構成対応）

7人制（村人×3 / 占い師×1 / 人狼×2 / 騎士×1 など）の編成を追加し、夜フェーズに**人狼同士だけが参加できる会話ターン**を追加する。

- 夜フェーズの構成：① 狼同士の非公開会話 → ② 狼が襲撃対象を決定 → ③ 占い師が占い → ④ 騎士が護衛先を決定
- 狼同士の会話は複数回（最大2〜3往復）できる
- 騎士は独話（他者との会話なし）で護衛対象を考える。占い師の独話と同じ扱い
- ログは非公開（spectator_logにのみ記録）
- 7人制に合わせてゲームエンジンの役職設定・勝利条件も拡張する

## ❌ 19. エージェントの性別・年齢・口調

現在の `Persona` は `style`（性格の文字列）・`lie_tendency`・`aggression` のみ。
キャラクターの個性をより豊かにするため以下を追加したい。

- `gender`: 性別（例: "male" / "female" / "none"）
- `age`: 年齢または年代（例: 17, "teen", "adult"）
- `speech_style`: 口調（例: "polite"（丁寧語）/ "casual"（普通）/ "blunt"（ぶっきらぼう））

プロンプトへの反映場所は `build_persona_prompt()`。
`speech_style` は特に日本語モードで効果が大きい（敬語・タメ口・荒い言葉遣い）。

## ❌ 18. memory_summary の自動要約

現在 `memory_summary` は追記のみで上限がなく、ゲームが長引くほどプロンプトが肥大化する。
一定件数を超えたらLLMに自己要約させてリストを圧縮する仕組みを入れたい。

- トリガー例：`memory_summary` が N 件（例: 20件）を超えたとき
- 処理：LLMに「この記憶リストを3〜5件に要約せよ」と指示し、返ってきた要約で置き換える
- 実装場所：`memory.update_memory()` の末尾でチェックするのが自然
- 要約呼び出しは `llm/client.py` に `call_memory_summarize()` として追加する
